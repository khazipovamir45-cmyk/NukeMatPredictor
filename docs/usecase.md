# Use Cases

Документ описывает основные сценарии использования системы **NukeMatPredictor**.

Структура каждого Use Case соответствует методическим требованиям курса:
**Название → Акторы → Предусловия → Основной поток → Альтернативные потоки → Исключения → Постусловия**.

Графическая модель Use Case (диаграмма) расположена в файле `usecase.png` (исходник — `usecase.drawio`).

## UC-1. Подбор материала под условия эксплуатации

### Название
Подбор материала под заданные условия эксплуатации.

### Акторы
- **Пользователь** (студент / инженер / научный сотрудник) — основной актор.
- **Система NukeMatPredictor** — веб-приложение на Flask.
- **ML-модель** — внутренний компонент, выполняющий kNN-поиск по характеристикам материалов.
- **База данных** — SQLite, таблица `materials` (200 записей).

### Предусловия
- Пользователь открыл сайт NukeMatPredictor в браузере.
- База данных материалов загружена и доступна.
- ML-модель инициализирована.

### Основной поток
1. Пользователь открывает главную страницу сайта.
2. Пользователь нажимает кнопку «Перейти к расчёту».
3. Система открывает страницу с формой ввода параметров.
4. Пользователь заполняет поля формы:
   - температура эксплуатации (°C);
   - доза облучения (dpa);
   - минимальная требуемая прочность (МПа);
   - теплопроводность (Вт/м·°C);
   - теплоёмкость (Дж/кг·°C).
5. Пользователь нажимает кнопку «Рассчитать».
6. Система валидирует введённые значения.
7. Система передаёт параметры в ML-модель.
8. ML-модель выполняет kNN-поиск по таблице `materials` и ранжирует материалы по близости к заданным условиям.
9. Система формирует список из 3 материалов со статусами `optimal` / `acceptable` / `unsuitable`.
10. Система строит график зависимости прочности от дозы облучения для оптимального материала.
11. Система строит график зависимости прочности от температуры.
12. Система отображает страницу результатов с таблицей материалов и графиками.
13. Пользователь анализирует результаты.

### Альтернативные потоки

**А1. Пустая форма**
1. На шаге 6 система обнаруживает, что одно или несколько обязательных полей не заполнены.
2. Система выделяет незаполненные поля и показывает предупреждение «Заполните все обязательные поля».
3. Сценарий возвращается к шагу 4 основного потока.

**А2. Некорректные значения**
1. На шаге 6 система обнаруживает значения вне допустимого диапазона (отрицательная температура, отрицательная доза и т. п.).
2. Система выделяет проблемное поле и показывает сообщение об ошибке с указанием допустимого диапазона.
3. Сценарий возвращается к шагу 4 основного потока.

**А3. Нет подходящих материалов**
1. На шаге 9 ML-модель не находит материалов, удовлетворяющих заданным условиям.
2. Система отображает сообщение «По заданным параметрам не найдено подходящих материалов. Попробуйте смягчить условия эксплуатации».
3. Пользователь возвращается к шагу 4 и изменяет параметры.

**А4. Изменение параметров на интерактивном графике**
1. На шаге 13 пользователь использует интерактивный слайдер температуры на графике.
2. Система пересчитывает значения и перерисовывает график без перезагрузки страницы.
3. Сценарий возвращается к шагу 13.

### Исключения

**И1. Ошибка базы данных**
1. Система не может получить доступ к таблице `materials` (БД недоступна, повреждена или не инициализирована).
2. Система отображает сообщение «Ошибка соединения с базой данных. Попробуйте позже».
3. Ошибка фиксируется в логе сервера.
4. Сценарий завершается без получения результата.

**И2. Ошибка ML-модели**
1. ML-модель выбрасывает исключение при обработке (некорректная структура данных, ошибка вычислений и т. п.).
2. Система логирует подробности ошибки на стороне сервера.
3. Пользователю показывается обобщённое сообщение «Ошибка расчёта. Обратитесь к администратору».
4. Сценарий завершается без получения результата.

### Постусловия
- Пользователь получил ранжированный список материалов с характеристиками и графиками.
- Графики зависимости свойств от условий эксплуатации построены и отображены.

## UC-2. Сохранение результатов расчёта

### Название
Сохранение результатов расчёта на устройство пользователя.

### Акторы
- **Пользователь** — основной актор.
- **Система NukeMatPredictor**.
- **Браузер пользователя** — выполняет загрузку файла.

### Предусловия
- Завершён сценарий UC-1: на экране отображены результаты расчёта.

### Основной поток
1. Пользователь нажимает кнопку «Сохранить результаты» на странице результатов.
2. Система формирует файл с введёнными параметрами, таблицей материалов и графиками.
3. Браузер инициирует загрузку файла на устройство пользователя.
4. Пользователь подтверждает сохранение и выбирает место для файла.

### Альтернативные потоки

**А1. Отмена сохранения**
1. На шаге 4 пользователь отменяет диалог сохранения в браузере.
2. Файл не сохраняется, страница результатов остаётся открытой.
3. Сценарий завершается.

### Исключения

**И1. Ошибка генерации файла**
1. На шаге 2 система не может сформировать файл (например, недоступны графики или закончилась память).
2. Система отображает сообщение «Не удалось сохранить результаты. Попробуйте ещё раз».
3. Сценарий завершается.

### Постусловия
- Файл с результатами расчёта сохранён на устройстве пользователя.
