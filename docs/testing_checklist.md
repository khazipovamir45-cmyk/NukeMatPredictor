# Чек-лист тестирования NukeMatPredictor

## 1. Проверка базы данных

- [ ] Файл `materials.db` создаётся
- [ ] Таблица `materials` существует
- [ ] Все поля присутствуют
- [ ] Данные загружаются из CSV
- [ ] Нет ошибок при загрузке

---

## 2. Проверка CSV

- [ ] Файл `nuclear_materials.csv` читается
- [ ] Все колонки корректны
- [ ] Нет пустых значений
- [ ] Типы данных корректны (float)

---

## 3. Проверка нормализации

- [ ] Создаётся `scaler.pkl`
- [ ] Создаётся `features.npy`
- [ ] Данные нормализованы (не все одинаковые значения)
- [ ] Размерность совпадает (N x 5)

---

## 4. Проверка подготовки данных

- [ ] Создаётся `materials_list.pkl`
- [ ] У каждого материала есть:
  - name
  - temp_coef
  - dose_coef
  - corrosion_rate
  - strength_ref

---

## 5. Проверка расчёта strength_ref

- [ ] Формула работает корректно
- [ ] Нет отрицательных значений
- [ ] Проверено вручную на 2–3 материалах

---

## 6. Проверка k-NN

- [ ] Загружаются данные (`load_knn_data`)
- [ ] Выполняется поиск (`find_nearest`)
- [ ] Возвращается 5 ближайших
- [ ] Расстояния отсортированы

---

## 7. Проверка ранжирования

- [ ] Есть 3 категории:
  - optimal
  - acceptable
  - unsuitable
- [ ] Есть статус и текст

---

## 8. Проверка Flask

- [ ] Главная страница открывается (`/`)
- [ ] Форма работает (`/form`)
- [ ] POST-запрос отправляется
- [ ] `/result` возвращает результат

---

## 9. Проверка экспорта CSV

- [ ] `/download_csv` работает
- [ ] Файл скачивается
- [ ] Данные корректны

---

## 10. Проверка крайних случаев

- [ ] Очень высокая температура
- [ ] Очень большая доза
- [ ] Нулевые значения
- [ ] Неверный ввод (обработка ошибок)
